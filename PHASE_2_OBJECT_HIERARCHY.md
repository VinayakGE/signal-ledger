# Phase 2 Object Hierarchy — Current State

**Date:** 2026-06-23

**Purpose:** Calibrate confidence levels for three emerging objects. Distinguish "signal" from "constraint."

---

## Tier 1: Most Mature Object

### Evidence Class Taxonomy

**Current Status:**
- Instrument (not hypothesis)
- Under validation (not yet constraint)
- Strongly supported (predictive claim + confirmation across 5 cases)

**The Prediction:**
Class C evidence (contemporaneous motivation, decision rationale at time T) should be systematically scarce in organizational case studies.

**The Confirmation:**
- Kodak: Class C blocked (defunct company archives)
- Nokia: Class C partially blocked (board privacy norms)
- WeWork: Class C uncertain (private company; not yet investigated)
- GE: Class C blocked (network policy)
- BlackBerry: Class C not found (primary source declined)

**Assessment:** Strong signal. Not yet constraint.

**Why not yet constraint:**
- Sample size (5 cases) is strong but not exhaustive
- Could be that Class C evidence exists but is systematically hidden for reasons we haven't identified
- Could be domain-specific to corporate/organizational failure (vs. policy, political, scientific failures)
- Prediction was "scarce," not "impossible" — leaves room for exceptions

**What would elevate to constraint:**
- Consistent scarcity across 10+ independent cases
- Verification that archives were searched exhaustively (not just unavailable)
- Clear mechanism explaining why Class C is systematically inaccessible (institutional privacy norms, destruction, etc.)

**Current role:** Measurement instrument. Explains evidence landscape. Guides expectations about what can and cannot be established in future investigations.

**Confidence:** High (signal). Medium (constraint status). Low (universality).

## Tier 2: Second Most Mature Object

### PP#002 (Observable Selection, Opaque Motivation)

**Current Status:**
- Pressure point (not hypothesis)
- Survived compression (found independently in Kodak, BlackBerry, GE)
- Awaiting adversarial test (pre-registered falsification conditions)

**How It Survived Compression:**
- BlackBerry: BBM proposal visible; motivation for rejection opaque ✓
- GE: Model divergence visible; Immelt's reasoning opaque ✓
- Kodak: Capital allocation visible; film-protection motivation opaque ✓

**Why It Needs Adversarial Test:**
- Found retrospectively in all three cases (pattern-matching after-the-fact)
- Not predictively validated (predicted → sought with falsification → found)
- Needs pre-registered test to distinguish real pattern from confirmation bias

**Falsification Condition (Pre-Registered for WeWork):**
If WeWork produces observable strategic choices (capital allocation, IPO timing, governance decisions) with contemporaneous Class C documentation explaining motivation, PP#002 is wounded.

**Current role:** Pressure point under validation. Candidate for promotion to constraint or topology after adversarial test.

**Confidence:** Medium (survived compression). Awaiting validation.

## Tier 3: Third Most Mature Object

### Zone A/B Boundary (Observable Structure vs. Opaque Motivation)

**Current Status:**
- Measurement instrument (not hypothesis)
- Compression-supported (appears in Kodak, BlackBerry)
- Partially derivative of PP#002 (similar structure, but distinct scope)
- Needs independent pressure

**How It Differs from PP#002:**
- PP#002: Focuses on selection between alternatives (choice between options)
- Zone A/B: Focuses on organizational behavior generally (not just alternatives)

**Zone B Examples:**
- Kodak: Margin collapse, R&D spending levels, organizational structure
- BlackBerry: Strategic announcements, product decisions, capital allocation

**Zone A Examples:**
- Kodak: Why film protection vs. digital acceleration chosen
- BlackBerry: Why Lazaridis supported killing BBM despite Mac-level recognition

**Why Derivative Status:**
PP#002 and Zone A/B are related but not identical. Every PP#002 case has a Zone A/B structure, but not every Zone A/B case necessarily has a PP#002 observable selection.

**What Would Give Independent Pressure:**
- Case that shows observable structure (Zone B) WITHOUT visible selection between alternatives (would establish Zone A/B exists beyond selection context)
- Or: Case that shows Zone A opacity in different context (leadership beliefs, market interpretation, strategic assumptions) not tied to alternative selection

**Current role:** Measurement framework. Explains why some questions cannot be answered from public evidence.

**Confidence:** Medium (compression-supported). Awaiting independent pressure.

## Critical Discipline for Validation

### No Object May Be Evaluated By Its Own Language

**Wrong (Object's Language):**

Does PP#002 appear in WeWork?

Evaluating: "Does the pattern exist?"

Risk: Yes/no becomes a matter of interpretation. The object gets to define what counts as success.

---

**Right (Reality's Language):**

Can WeWork produce observable strategic choices alongside transparent contemporaneous explanation of motivation?

Evaluating: "Does reality match the object's requirements for survival?"

Risk: Minimized. Reality either has the explanation or doesn't.

---

### Alternative Example

**Wrong (Object's Language):**

Does Zone A/B boundary map to WeWork?

Risk: "Observable structure + opaque motivation" is vague. Object gets to decide what counts as "observable" and "opaque."

---

**Right (Reality's Language):**

In WeWork, can we document Neumann's strategic choices (capital rounds, timing, governance) with contemporaneous Class C explanation of his reasoning?

Risk: Minimized. Either Class C documentation exists or it doesn't.

## The Shift That Signals Phase 2 Is Real

**Phase 1 Question:**
How do we strengthen this hypothesis?

**Phase 2 Question:**
How do we kill this?

The research program has inverted its burden of proof.

Objects must earn survival through adversarial testing, not through pattern-recognition.

## The Repository's Actual Achievement

Not the objects themselves.

But the explicit structure:

**How to wound an object:**

1. **Evidence Class Taxonomy:** Find Class C documentation abundant in case where prediction said it should be scarce
2. **PP#002:** Find contemporaneous explanation of observable choice (Class C document explaining Neumann's motivation)
3. **Zone A/B Boundary:** Find case with observable choices but transparent contemporaneous motivation

**How to strengthen an object:**

1. **Evidence Class Taxonomy:** Repeated confirmation of Class C scarcity across diverse cases
2. **PP#002:** Observable choices with opaque motivation in independent case, without deliberate hunting
3. **Zone A/B Boundary:** Structural pattern holding in case that wasn't designed to test it

This is disciplined research. Not confirmation-seeking.

## Next Phase: WeWork as Adversarial Test

Investigators must enter with explicit wound-seeking stance:

**Not:** "Let's see if PP#002 appears"

But: "Let's find what would wound PP#002 and see if it's there"

**Not:** "Does the pattern hold?"

But: "What would prove the pattern false?"

If investigators find themselves defending an object instead of attacking it, the investigation has failed.

The object should earn survival only by surviving genuine adversarial pressure.
