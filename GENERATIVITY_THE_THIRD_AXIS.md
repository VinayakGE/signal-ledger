# Generativity — The Third Axis

**Date:** 2026-06-23

**The Hazard:** A research program can become methodologically self-admiring.

Once it develops pre-registration, falsifiers, wound conditions, and compression passes, it can start protecting the methodology instead of using it to learn.

---

## The Hazard Illustrated

### Object That Survives But Doesn't Predict

**Statement:** "Organizations sometimes hide their motivations."

**Strength:** Low-to-medium
**Survival:** Very high (will survive almost any attack)
**Generativity:** Very low

**Why survival is illusory:**
The statement is vague enough to accommodate almost any observation. Organizations hiding motivations? Yes. Organizations disclosing motivations? Also consistent (they chose to disclose, which is a form of organizational behavior).

**Why it doesn't generate:**
It doesn't tell you where to look, what to expect, how to be surprised. It's purely descriptive.

---

### Object That Survives And Predicts

**Statement:** "Class C evidence (contemporaneous decision rationale) should be systematically scarce in organizational case studies because: defunct companies don't preserve archives, public companies maintain board privacy, private companies may not document decisions, primary sources actively protect disclosure."

**Strength:** High
**Survival:** Medium (can be wounded if Class C is abundant)
**Generativity:** High

**Why survival is meaningful:**
The statement is specific. It makes predictions. It can fail.

**Why it generates:**
It tells investigators:
- What to search for (Class C documents)
- What not to expect (abundant contemporaneous rationale)
- How to be surprised (if Class C is abundant, prediction fails)
- How to move forward (if prediction holds, expect Zone A opacity in future cases)

---

## The Third Axis: Generativity

**Question:** Does this object produce new predictions?

**Scale:**
- High: Generates search strategy; enables surprise; produces falsifiable predictions
- Medium: Organizes existing observations; hints at predictions; partial search guidance
- Low: Purely descriptive; accommodates observations; no predictive power

**What it measures:** Instrumental value (not just explanatory value)

**Why it matters:** An object that survives but generates nothing is not accumulating knowledge; it's accumulating vocabulary.

## The Critical Question for WeWork

**Not just:** Can we wound PP#002?

**But:** Did PP#002 tell us where to look before we looked?

**Example of strong generativity:**

Evidence Classes predicted:
- Look for contemporaneous decision documents (Class C)
- Don't expect them to be abundant
- If abundant, our prediction fails

Investigators search with this expectation in mind. They find:
- WeWork founder's private journals (Class C, abundant)
- Evidence Classes prediction wounded
- Learning: Class C scarcity may be institutional, not universal

Investigators changed search strategy based on object's prediction. Object enabled surprise.

---

**Example of weak generativity:**

PP#002 predicts... what exactly?

Without clear prediction, investigators search without guidance. They find:
- Some observable choices with opaque motivation
- Some transparent motivation
- Object survives because it's vague enough

Did PP#002 tell us where to look? Unclear. Did it enable surprise? Only if we had expected something specific, which we didn't.

## What Distinguishes Resilient from Predictive

### Resilient Object

- Survives attacks
- But survives because it's vague
- Doesn't guide search
- Can't be surprised
- Accommodates observations after-the-fact

**Risk:** Becomes a vocabulary without content

---

### Predictive Object

- Makes specific predictions
- Can fail (falsifiable)
- Guides search strategy
- Enables surprise (can learn why prediction failed)
- Generates next investigation

**Benefit:** Accumulates knowledge, not just vocabulary

---

## The Threshold That Matters

A disciplined repository ≠ An accumulating research program

**Disciplined but not accumulating:**
- Pre-registration? Yes
- Falsification conditions? Yes
- Compression passes? Yes
- But: Objects are vague; nothing generates new investigations
- Result: Beautiful methodology, stalled learning

---

**Disciplined and accumulating:**
- Pre-registration? Yes
- Falsification conditions? Yes
- Compression passes? Yes
- And: Objects make predictions; generate search strategies; enable surprise
- Result: Methodology serves learning; learning guides methodology

## The Guard Rail Against Methodological Self-Admiration

### Questions That Signal Health

- "What does this object predict?"
- "Did it guide our search?"
- "Were we surprised by what we found?"
- "Would the next investigator know where to look based on this object?"

### Questions That Signal Drift

- "Is the methodology rigorous?"
- "Are the falsification conditions well-defined?"
- "Does the object survive attacks?"

The first set focuses on accumulation. The second focuses on protection.

Both matter. But if the second set dominates, the program has inverted its priorities.
