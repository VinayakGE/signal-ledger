## The Three Outcomes To Look For

### 1. Wounds

An object fails its pre-registered falsification condition.

Example: "Class C evidence is abundant" (wounds Evidence Class Taxonomy)

**Value:** Medium (teaches domain-specificity)

**Action:** Document wound; mark object as domain-limited

### 2. Survivors

An object survives its pre-registered test.

Example: "PP#002 appears independently in WeWork" (survives)

**Value:** Medium (confirms robustness)

**Action:** Upgrade confidence; carry forward

### 3. Surprises

Evidence reveals something not predicted by any object.

Example: "Neumann's conviction changed mid-stream in ways not captured by existing frameworks"

**Value:** High (highest-information outcome)

**Action:** Stop and reassess; revise frameworks; determine if new object needed

## When To Create New Objects

**New objects enter the repository only when:**

1. Evidence reveals something existing objects don't capture
2. The something is:
   - Specific (not vague)
   - Falsifiable (can be wounded)
   - Generative (produces predictions)
   - Necessary (existing objects insufficient)

**Until then:** Use existing frameworks.
